# Мобильный туристический гид с модулями геймификации и автоматической модерации контента (TouristGuideApp)

Программный комплекс представляет собой систему мобильного путеводителя по достопримечательностям региона, включающую подсистемы автоматической лингвистической фильтрации пользовательского ввода на базе нейросетевых моделей и игровые механики стимулирования активности пользователей (геймификацию).

Система состоит из двух взаимосвязанных компонентов:
1. **Мобильное клиентское приложение (Android)** — реализовано на языке Kotlin с использованием архитектурного шаблона MVVM.
2. **Микросервис автоматической модерации и семантической валидации контента** — реализован на языке Python с использованием веб-фреймворка FastAPI, библиотеки SQLAlchemy (для работы с СУБД PostgreSQL) и NLP-моделей для фильтрации контента и выявления дубликатов.

---

## Функциональные подсистемы

### 1. Мобильное клиентское приложение (Android)
* **Интерактивный картографический модуль:** Визуализация достопримечательностей на базе библиотеки OpenStreetMap (osmdroid) с поддержкой локального кэширования тайлов для автономной работы без доступа к сети Интернет.
* **Каталог объектов историко-культурного наследия:** Структурированный список мест с поддержкой поиска, фильтрации по категориям, а также детальными карточками объектов (историческая справка, аудиогид, галерея изображений).
* **Модуль пользовательского контента:** Возможность публикации текстовых отзывов, выставления оценок, загрузки фотографий и отправки жалоб на некорректный контент.
* **Модуль навигационных маршрутов:** Отображение готовых экскурсионных маршрутов с визуализацией путевых точек и траектории движения на интерактивной карте.
* **Модуль геймификации:** 
  * Профиль пользователя с системой начисления очков опыта и расчета уровней.
  * Система достижений (награды за посещение объектов и написание отзывов).
  * Динамические еженедельные задания для вовлечения пользователей.

### 2. Подсистема автоматизированной модерации контента (filter_service)
* **Лингвистическая фильтрация отзывов:**
  * Анализ текстового ввода на предмет наличия спама, нежелательных веб-ссылок, коммерческой рекламы и аномального использования верхнего регистра (CapsLock).
  * Выявление нецензурной лексики с использованием специализированного словарного фильтра.
  * Классификация текста на токсичность и оскорбления с использованием предобученной NLP-модели (RuBERT).
* **Семантический контроль дублирования:**
  * Выявление семантически близких отзывов с помощью векторных представлений слов (эмбеддингов Navec).
  * Контроль уникальности добавляемых туристических объектов по геодезическим координатам (расчет ортодромического расстояния по формуле Хаверсина).

## Архитектура и стек технологий

### Мобильный клиент (Android):
* **Язык программирования:** Kotlin 1.9+
* **Архитектурный шаблон:** MVVM (Model-View-ViewModel), чистая архитектура (слои Data, Domain, UI), инверсия зависимостей (DI через `AppContainer`).
* **СУБД:** 16 таблиц (пользователи, объекты, отзывы, маршруты, задачи, достижения, журнал активности).

### Микросервис модерации:
* **Язык программирования:** Python 3.10+
* **Фреймворк:** FastAPI (интеграция через ASGI-сервер Uvicorn).
* **СУБД & ORM:** PostgreSQL + SQLAlchemy + asyncpg (асинхронный драйвер базы данных).
* **Инструменты NLP и анализа данных:** russian-bad-word-censor, bert_classifier_rubert_tiny_toxicity , Navec.

## Руководство по развертыванию и запуску

### 1. Мобильное клиентское приложение (Android)
1. Импортируйте корневую директорию `TouristGuideApp` в среду разработки **Android Studio**.
2. Выполните синхронизацию проекта с помощью сборочной системы Gradle.
3. Подключите физическое Android-устройство (минимальная версия Android 8.0 / API 26) с включенной отладкой по USB или запустите виртуальный эмулятор.
4. Выполните сборку и запуск приложения с помощью кнопки **Run**.
*Приложение инициализируется с предзаполненной демонстрационной базой данных SQLite для демонстрации функционала.*

### 2. Микросервис модерации (Python/FastAPI)
1. Перейдите в каталог микросервиса:
   ```bash
   cd filter_service
   ```
2. Установите программные зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Загрузите предобученные векторные представления Navec [navec_hudlit_v1_12B_500K_300d_100q.tar](https://storage.yandexcloud.net/natasha-navec/packs/navec_hudlit_v1_12B_500K_300d_100q.tar) и сохраните файл в каталоге `filter_service`.
4. Настройте параметры подключения к СУБД в конфигурационном файле `config.py` или файле переменных окружения `.env`.
5. Разверните структуру таблиц в целевой СУБД PostgreSQL, применив SQL-скрипт `db/create_database.sql`.
6. Запустите сервер приложений:
   ```bash
   uvicorn main:app --reload
   ```
